# 自研重构指南

## 这个词是什么意思

Form-A 的社区版代码在 GitHub 上完全公开。但作为产品它不值什么钱——值钱的是 **为什么那么写的决策逻辑**。

客户买 Form-B，是因为他们看懂之后，可以用自己的技术栈重写。

真实场景：

> 某集成商 CTO：「这docker-compose跑得好好的我为什么要买？我让手下三个月照着思路用Go重写一遍，做成我们的产品卖终端客户。」

这就对了。这就是Form-B的价值——他花的三个月，看完这份指南可以缩到三周。他花了¥2,980，省了9个人周的调研成本。

## 本指南解决的问题

| 问题 | 指南怎么帮 |
|------|-----------|
| n8n的工作流怎么设计才能稳定？ | 给最佳实践模板和坑点清单 |
| PostgreSQL高可用怎么搭？ | 给Patroni+etd详细配置和脑裂处理 |
| 授权怎么防止被破解？ | RSA签名+机器指纹的原理和实现 |
| 分销分账怎么做？ | 数据库设计+API设计 |
| K8s迁移注意什么？ | 服务发现、配置管理、日志聚合 |
| 用Go重写各组件要多久？ | 逐组件工时估算 |

## 格式

- 一篇完整的 Markdown 技术文档（约100页印出来）
- 附代码片段（伪代码，说明逻辑即可，不绑定具体语言）
- 附架构图（Mermaid，可直接渲染）
