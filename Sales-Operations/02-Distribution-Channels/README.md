# 分发渠道总览

> 覆盖逻辑：开发者 → 企业决策者 → 代理商

## 渠道矩阵

| 渠道 | 目标 | 优先级 | ROI |
|------|------|-------|-----|
| GitHub | 开发者引流、社区版分发 | ⭐⭐⭐⭐⭐ | 自然流量，几乎零成本 |
| 微信生态 | 中国市场企业客户触达 | ⭐⭐⭐⭐ | 高信任，高转化 |
| 代理商 | 区域/行业覆盖 | ⭐⭐⭐⭐ | 杠杆，放大销售半径 |
| 官网 | 商业化转化、产品展示 | ⭐⭐⭐ | 品牌资产，长期投入 |
| 技术社区 | 口碑传播、案例沉淀 | ⭐⭐⭐ | 慢热但长效 |
| B 站/知乎 | 视频 & 文章获客 | ⭐⭐ | 内容成本高 |

## 渠道漏斗

```
GitHub Star/Fork/Clone
    ↓
README 中企业版引导
    ↓
官网 / 公众号
    ↓
联系销售 / 申请试用
    ↓
License 交付 → 代理商分账
```

## 当前重点

1. **GitHub**：社区版是流量入口，README 即官网
2. **微信生态**：公众号 = 内容获客，微信群 = 私域运营
3. **代理商**：优先签 5~10 个核心代理商覆盖主要城市

## 数据追踪

每个渠道配置独立追踪参数：
- GitHub: README 中的企业版体验链接带 `?source=github`
- 微信公众号: 菜单栏链接带 `?source=wechat`
- 代理商: License 中的 `referrer_code` 字段
