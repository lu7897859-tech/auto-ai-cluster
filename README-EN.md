## 🧠 Core Philosophy

**Survival-First Principle**: In resource-constrained environments, core business flow always takes priority. Circuit breaker releases non-critical services by priority, ensuring n8n + PostgreSQL never go down.

**Minimal Logic Principle**: Every component must fit its architecture diagram on a single A4 sheet. System survival probability ∝ 1 / (modules × interfaces × states).

**2C4G Constraint-Driven Design**: Extreme constraints produce better decisions. The brutal 2C4G limit forces each service to do exactly one thing — avoiding the bloat of modern AI infrastructure.
