# ✅ 代理商演示前检查清单

> **黄金法则：** 你在客户面前演示之前，自己先跑三遍。任何演示事故=你丢单。

---

## □ 1. 环境准备（至少提前1天）

- [ ] 准备好一台干净的演示服务器（推荐 8C16G+，Ubuntu 22.04）
- [ ] 在自己的演示机上从头跑一遍 `deploy.sh`，**确保一次性通过**
- [ ] 记录部署过程中的任何异常和解决方法（加到 FAQ 里）
- [ ] 确认演示环境的 IP/域名是客户能访问的
- [ ] 关闭防火墙或开放必要端口（记录开放的端口列表）
- [ ] 确认演示环境有互联网访问权限（不需要的请在离线环境下也测试一遍）

> **🚨 必记：** 不要在客户面前跑 `deploy.sh`。演示室网络不稳定就翻车。提前部署好，演示时只点「下一步」。

---

## □ 2. 工作流模板准备

- [ ] 提前和客户沟通他们的业务场景
- [ ] 准备好2~3个和客户行业相关的 Agent 工作流模板
- [ ] 预置好模板的输入/输出参数
- [ ] 至少有一个「从输入到输出全自动」的端到端 Demo 场景
- [ ] 准备1个「可以在演示现场改参数」的灵活场景（让客户参与）

> **场景示例：**
> - 电商行业：自动客服分流 + 订单异常处理
> - 金融行业：信息聚合 + 合规审查 Workflow
> - 制造行业：设备告警联动处理
> - 通用场景：内容审核 + 自动回复

---

## □ 3. License 生成演示

- [ ] 确认已在 License 管理系统中注册为代理商
- [ ] 准备好生成演示 License 的流程截图或操作视频
- [ ] 现场演示：从登录到 License 生成的完整链路（控制在 3 分钟内）
- [ ] 准备好不同时长（7天/30天/90天）的演示 License
- [ ] 演示 License绑定机器指纹的方式
- [ ] 演示 License 到期后的行为（降级为社区版，数据不丢失）

---

## □ 4. 可视化 UI 演示

- [ ] 打开集群总览页面（Agent 在线状态 / 资源消耗 / 任务队列）
- [ ] 打开工作流编辑器（拖拽式/配置式）
- [ ] 展示一个正在运行的工作流（实时日志）
- [ ] 展示 Agent 之间的通信交互
- [ ] 展示告警和历史记录

---

## □ 5. 性能/稳定性演练

- [ ] 准备一个压力测试脚本（同时跑 10+ Agent 任务）
- [ ] 展示任务队列管理（并发控制、优先级、限流）
- [ ] 展示异常 Agent 自动恢复
- [ ] 在演示过程中可以「人为制造故障」展示自动恢复（可选，但效果极好）

---

## □ 6. FAQ 快速应对

- [ ] 复习 `customer-faq.md` 前 20 个最常见问题
- [ ] 准备你自己的「答不上来怎么办」应急方案：「我先记下来，24小时内给你书面回复」
- [ ] 准备好「你怎么证明……」这类问题的实物证据

---

## □ 7. 报价和签约准备

- [ ] 打开 `commission-calculator.md` 算好客户可能需要的方案报价
- [ ] 准备好不同版本的报价单（社区版/企业版/源码授权+单独组件）
- [ ] 准备好电子版合同模板
- [ ] 准备好支付方式说明（对公转账/在线支付/预充值）

---

## □ 8. 演示日最终确认

- [ ] 网络：用客户现场的网络访问一次演示环境
- [ ] 投影/投屏：确认显示效果，字体大小调好
- [ ] 声音：如果用语音解说，麦克风准备好
- [ ] 备机：如果笔记本翻车，手机热点能连上演示环境
- [ ] 电量：笔记本 + 手机 + 充电宝 全部满电
- [ ] 静音：手机全程静音
- [ ] 录屏：打开本地录屏，防止演示完之后客户想复看
- [ ] 时间：控制总时长 30~45 分钟，留 15 分钟问答

---

## □ 9. 演示后的行动

- [ ] 当场发演示回放链接（如果录了屏）
- [ ] 整理客户提出的问题和我们的答复（邮件发给他）
- [ ] 跟进时间：**24小时内** 电话回访
- [ ] 更新 CRM 中的客户状态

---

## ⚠️ 演示大忌

| ❌ 不要做 | ✅ 要这样做 |
|-----------|------------|
| 现场敲代码 | 提前准备好，演示时只点鼠标 |
| 现场部署 | 提前部署好 |
| 说「这有点复杂我解释一下」 | 说「我们看一下这个效果」 |
| 背话术背不出来 | 把话术打印出来放桌上 |
| 贬低竞品 | 说「XX也不错，但我们XXX更好」 |
| 说超出产品能力的话 | 说「这个我们可以确认一下再回复你」 |
| 对自己产品不熟的问题编答案 | 说「这个问题我专业的部分先回答，其余部分我给你书面回复」 |

---

> **记住：** 客户买的不是技术，是信任。一个流畅的演示 = 信任的第一步。
