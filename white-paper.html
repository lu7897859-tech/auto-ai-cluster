📋 Executive Summary

This white paper documents the complete engineering journey of building a production-grade autonomous AI cluster under extreme resource constraints — 2 vCPUs and 4GB RAM (a $5/month VPS).

Instead of starting with "what's the industry standard?", every decision is grounded in first principles: What's the physical minimum? What can we eliminate? What breaks first under load?

The result is a 7-service Docker Compose cluster — n8n, PostgreSQL, Redis, Nginx, AI-API Gateway, Authorization Server, pgAdmin — that deploys with a single command and survives production traffic.

n8n's memory footprint (~80MB idle) was the decisive factor. At 2C4G, every MB matters.

Survival-First Circuit Breaker

Priority-based resource release under memory pressure:

  • Tier 1 (Never kill): n8n + PostgreSQL
  • Tier 2 (Graceful degrade): Redis cache
  • Tier 3 (Rate limit → queue → fail open): AI-API gateway
  • Tier 4 (Nice to have): Monitoring, pgAdmin

Authorization System

RSA-2048 key pair + machine fingerprint binding. 998 lines of Python, 6 API endpoints, all tested. Supports distributor ID embedding and activation reporting. 7-day offline cache for intermittent connectivity.
